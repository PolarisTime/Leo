# 数据库备份与恢复职责

Leo 应用不提供数据库备份、导入、导出或备份文件下载能力，也不会在应用进程中执行 `pg_dump`、`psql` 或数据库恢复任务。

生产数据库的备份与恢复必须由数据库平台、云厂商托管能力或独立运维任务负责。运维方案至少应明确：

- 备份周期、保留周期和异地副本策略；
- PostgreSQL WAL 归档和时间点恢复能力；
- 备份加密、访问控制和审计记录；
- 定期恢复演练及恢复时间目标；
- 应用发布与数据库迁移失败时的恢复负责人。

数据库恢复不得通过 Leo API 或应用容器执行。恢复完成后，应先验证 Flyway 历史、核心表数量和应用只读探活，再恢复业务流量。
